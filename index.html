I always dabbled in web design as a tyke. I can still remember my geocities page and all the atrocities the late 90s allowed you to commit. I remember how many times I broke the family computer by messing around networking settings a 9 year old probably had no business getting into. I couldn't help myself. My mom had bought a huge 'Teach Yourself Windows 95' book and I had taken to that as my gospel. I read it cover to cover and boy, did I act on it.

Granted, everything I knew from heart was outdated but it provided a very solid foundation for learning. I was able to draw comparisons and see the evolution internet based technology had made in the last two decades. For example, my hobby of setting up virtual machines to run old operating systems became a legitimate skill! Who knew that doing that was the way to run your own hosting server? Instead of installing Windows 2000, though, I simply had to install a Linux Distribution. And now I had to learn Linux? No problem, I was already familiar with terminal commands under MS-DOS because that's where I played my video games.

I'm new to the industry, but I feel like I've been in it since a kid. I am now confident that this is what I want to do and where I want to grow as a professional. I'm ever adapting as this industry is ever changing and simply put: I am in love with learning.

I had never considered it a career until a beloved relative of mine sat me down and presented me with the option. I immediately jumped into the offer and soon found myself in a certification course for full-stack web development. It was absolutely wonderful! I felt the rush of my childhood dreams with each new skill I learned and each old skill refined. I finally found a purpose!

Python Reader for PSQL Data
Previous Next
Python PSQL

This is one of my favorite projects because I found out I really, really like organizing and reading SQL data. I was given an SQL file that contained 11.9mb of data with the task of answering three questions with it. I had to answer these multiple part questions by using a single query. I figured out how many steps it would take to answer different parts of the same question, created all the VIEWs needed for them, and then switched on over to Python. In Python, I wrote the functions that would print out the query results and had it each question answer print out in an easy to read table.

SQL Python Bootstrap Apache2

Like I've said before, I live for SQL databases so I started this small blogsite to keep track of characters and diary entries in a DnD campaign I'm part of. I wrote it in Python Flask with password hashing provided by passlib and itsdangerous. There's also basic authentication provided by Flask HTTP-AUTH. A DJango module handles file uploads and restricts them based on size/file-extension. I make use of cookies and keys as well. Then I threw it all up on an Ubuntu instance using Apache 2 via Amazon Web Services.

I did this in a few days off and on so there's a long TO-DO list to really polish it. It's a good learning experience for me!
